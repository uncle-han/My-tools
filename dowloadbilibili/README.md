# Bilibili 视频下载工具

基于 `yt-dlp` 的 Bilibili 批量视频下载脚本，支持自动重试、字幕嵌入、缩略图生成和失败重试机制。

## 功能介绍

- **批量下载**：从 `list` 文件读取多个视频链接，逐个下载
- **自动重试**：下载失败自动重试 2 次，间隔 10 秒
- **字幕嵌入**：自动下载中文自动字幕并嵌入到 MKV 视频中
- **缩略图生成**：下载完成后自动提取视频第一帧作为缩略图（.jpg）
- **ffmpeg 智能检测**：支持配置路径、PATH 环境变量、脚本同目录等多种方式查找 ffmpeg
- **文件占用等待**：下载完成后等待文件释放再生成缩略图，避免文件锁冲突
- **自动归档**：成功下载的链接归档到 `list_history_时间戳`，失败链接保留在 `list` 中方便重试

## 配置项说明

脚本顶部为配置区，共有 4 个配置项：

| 配置项 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| `DOWNLOAD_DIR` | string | 是 | 视频下载保存目录 | `r"E:\bilibili"` |
| `LIST_FILE` | Path | 否 | 视频链接列表文件路径，默认为脚本同级 `list` 文件 | - |
| `FFMPEG_PATH` | string | 否 | ffmpeg 可执行文件完整路径，留空则自动从 PATH 或脚本目录查找 | `r"C:\...\ffmpeg.exe"` |
| `COOKIES_BROWSER` | string | 否 | 浏览器名称，用于从浏览器读取 Cookie 登录 B 站，留空则不使用 Cookie | `"chrome"` / `"edge"` |

### 配置示例

```python
# ==================== 配置区 ====================
DOWNLOAD_DIR = r"E:\bilibili"
LIST_FILE = Path(__file__).parent / "list"
FFMPEG_PATH = r"C:\path\to\ffmpeg.exe"
COOKIES_BROWSER = ""  # 留空不使用 Cookie，或填 "chrome"/"edge" 等
# ===============================================
```

### 注意事项

- **ffmpeg**：不配置也能运行，但无法合并音视频和生成缩略图
- **Cookie**：如果需要下载会员视频或大会员画质，需配置浏览器名称
- **list 文件**：每行一个视频链接，`#` 开头的行为注释

## 依赖

- Python 3.x
- yt-dlp：`pip install yt-dlp`
- ffmpeg：[下载地址](https://ffmpeg.org/download.html)

## 使用方法

1. 在 `list` 文件中添加要下载的视频链接（每行一个）
2. 运行脚本：`python dowload.py`
3. 等待下载完成，视频保存在 `DOWNLOAD_DIR` 目录

## 输出文件格式

下载完成后，每个视频会生成：

- `视频标题.mkv`：合并后的视频文件（含内嵌字幕）
- `视频标题.jpg`：视频第一帧缩略图

## 失败重试机制

下载失败的链接会自动保留在 `list` 文件中，下次运行脚本时会继续尝试下载这些链接。历史成功记录会归档到 `list_history_YYYYMMDD_HHMMSS` 文件中，方便追溯。
